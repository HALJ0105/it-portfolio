Help Desk Lab

Built a virtual Help Desk environment in VMware to simulate real-world IT support scenarios, covering OS troubleshooting, network configuration, Active Directory account management, RSAT tools, and Group Policy enforcement.

VMware Workstation Player Windows Server 2019 Windows 11 Pro x2 Active Directory GPMC / RSAT

Lab Environment & Setup

  • VMware Workstation Player hosting all operating systems and servers used in the lab.
  • Two Windows 11 Pro VMs configured as client machines / end users.
  • Windows Server 2019 configured as the primary domain controller and server administrator.
  • Network set up via DHCP, with a dedicated DNS server and Active Directory services.
  • Tools used: Active Directory Users and Computers, Group Policy Management Console, Event Viewer, Remote Desktop, Command Prompt.

Scenarios Completed

Scenario 1 — Network Configuration

Objective: Configure and establish network connectivity between users on a shared DNS.

  1. Navigated to network settings on the domain controller, switched to static IP, and set the address to 10.1.10.2 with DNS address 10.1.10.1.
  2. Configured static IPs on both Windows 11 clients — 10.1.10.3 and 10.1.10.4.
  3. Confirmed IP address changes across all three machines using ipconfig.
  4. Tested connectivity between the domain controller and clients using ping.

Result: Domain controller and both clients successfully connected on the same server via static IP addressing.

Figure 1. Successful ping test confirming network connectivity between the domain controller and both Windows 11 clients.

Scenario 2 — Password Reset & Account Unlock

Objective: Resolve a scenario where a user is locked out of their account, then reset their password in Active Directory.

  1. Opened Active Directory Users and Computers on the domain controller (Server 2019).
  2. Located the user within their Organizational Unit (OU).
  3. Checked account status and unlocked the account.
  4. Reset the password and applied "User must change password at next logon."

Result: User successfully logged back in with the reset password.

Figure 2. Active Directory Users and Computers used to unlock a user account and reset the user's password.

Scenario 3 — Applying Group Policy for User Restrictions

Objective: Restrict Task Manager access for all domain users.

  1. Opened Group Policy Management Console.
  2. Created a new GPO to block Task Manager access.
  3. Linked the GPO to the "Users" OU.
  4. Forced a policy update (gpupdate /force) and tested it on a client machine.

Result: Task Manager was successfully blocked; the policy applied and enforced as intended.

Figure 3. Group Policy Object linked to the Users Organizational Unit to disable Task Manager access for domain users.

Skills Gained

  • Active Directory account creation & group management
  • GPO creation and enforcement
  • Network troubleshooting and configuration (ping, ipconfig, DNS fixes)
  • Remote desktop support & tools
  • RSAT tools installed and configured for remote domain administration

Key Takeaway

These three scenarios reflect the most common Tier 1 help desk tickets — connectivity issues, account lockouts, and policy enforcement — and strengthened both my technical troubleshooting process and my ability to document each step clearly for handoff or review.
